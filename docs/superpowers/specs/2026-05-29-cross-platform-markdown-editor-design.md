## 1. 개요 (Vision)

상용 마크다운 도구(특히 **Typora**)를 대체할 만한, **성능·UI/UX·유려한 디자인**을 모두 갖춘
크로스플랫폼 마크다운 **뷰어-에디터**를 만든다.

핵심 정체성은 **"집중형 라이터 + 인라인 WYSIWYG"**:
한 문서에 몰입하는 아름다운 타이포그래피, 입력 즉시 그 자리에서 서식으로 변환되는 편집 경험,
그리고 Mermaid·수식·코드 하이라이트 같은 풍부한 콘텐츠의 매끄러운 렌더링.

Rust를 **코어(시스템·성능 계층)** 로 채택하되, 인라인 WYSIWYG라는 유일한 난제는
검증된 웹 에디터 엔진으로 해결한다 (Tauri 2 = Rust 코어 + 웹뷰 프론트).

---

## 2. 타깃 사용자 & 대체 대상

- **대체 대상:** Typora / iA Writer / Bear 계열의 "집중형 라이터".
- **사용자:** 마크다운으로 글·노트·문서를 쓰는 개인. 한국어 사용자를 1급으로 고려(한글 타이포그래피·IME·CJK).
- **명시적 비(非)대상:** Obsidian/Logseq(지식 베이스·그래프·백링크), Notion/Anytype(블록 DB·협업).
  이들은 별개의 더 큰 제품군이며 v1 범위가 아니다.

## 5. 아키텍처

### 5.1 2-프로세스 모델 + 신뢰 경계
Tauri 2 앱은 단일 네이티브 바이너리(~수 MB) 안에서 두 영역이 IPC로 통신한다.

```
┌──────────── Tauri 2 앱 (Win · macOS · Linux) ────────────┐
│  Webview Frontend (TS)        ⇄ IPC ⇄        Rust Core    │
│  · CodeMirror 6 Live Preview         · 파일 I/O(원자적 저장) │
│  · markdown-it+KaTeX+Mermaid+Shiki   · 폴더 감시(notify)    │
│  · 커맨드 팔레트 · 파일트리 UI · 테마   · 전문 검색(tantivy)   │
│  (UI · 편집 · 렌더 전담)              · 워크스페이스/설정      │
│                                      · 내보내기 · 이미지 저장 │
│                                      · 네이티브 메뉴/다이얼로그│
└──────────────────────────────────┬───────────────────────┘
                                    │  (Rust만 FS 접근 = 신뢰 경계)
                            ┌───────▼────────┐
                            │  파일 시스템      │
                            │ vault/*.md       │
                            │ assets/          │
                            │ config.json      │
                            └─────────────────┘
```

**핵심 원칙:** 파일 시스템은 **Rust 코어만** 접근한다. 프론트(웹뷰)는 모든 FS 작업을 IPC 커맨드로 요청.

### 5.2 책임 분담
- **Rust 코어:** 파일 열기/저장(원자적: temp 쓰고 rename), 폴더 스캔 + 변경 감시(notify),
  전문 검색 인덱스(tantivy, 백그라운드 빌드), 워크스페이스/설정 영속화, 내보내기 오케스트레이션,
  이미지/asset 저장(해시 파일명), 네이티브 메뉴·다이얼로그, 대용량/비동기 IO.
- **웹뷰 프론트:** 에디터 UI 전체(CM6 + Live Preview 디코레이션), 마크다운 렌더(미리보기 위젯 + 내보내기 HTML),
  커맨드 팔레트, 파일 트리 UI, 설정/테마 UI.

### 5.3 IPC
- Tauri 커맨드(`#[tauri::command]`) + 이벤트.
- **타입 안전:** `tauri-specta`(+ `specta`/`ts-rs`)로 Rust 타입에서 TypeScript 바인딩 자동 생성.
- 커맨드 표면(초안): `open_folder`, `read_file`, `write_file`, `watch_start/stop`, `search`,
  `save_asset`, `export_html`, `export_pdf`, `load_settings`, `save_settings`.
- 이벤트(초안): `file_changed`(외부 수정), `index_progress`, `watch_event`.

### 5.4 렌더러 단일화 (드리프트 방지)
- CM6의 내부 파서(**Lezer markdown**)는 *편집 중 무엇을 숨기고/렌더할지*(인라인 디코레이션·활성 줄 원본 노출)만 결정.
- 실제로 그려지는 HTML(블록 위젯 + 내보내기)은 **항상 단일 파이프라인**(markdown-it + KaTeX + Mermaid + Shiki)에서 생성.
- 결과: **화면에 보이는 것 = 내보낸 결과**가 정확히 일치.

## 6. 문서 / 파일 모델

- **워크스페이스 = 폴더**(라이트 vault). 파일 트리는 Rust가 스캔해 제공.
- **활성 문서 = 한 번에 하나**(집중형 라이터 철학). 사이드바에서 다른 파일 클릭 시 활성 문서 교체(미저장 변경 가드). 탭은 v1.1.
- **진실의 원천(source of truth) = CM6 버퍼의 마크다운 원문.** 저장 = 버퍼를 디스크에 원자적 쓰기.
- **자동저장:** 디바운스(무입력 ~800ms) + 창 blur/hide 시. 저장 성공 시 "마지막 저장 콘텐츠 해시" 갱신(외부 변경 비교용).
- **이미지/asset:** `assets/`에 콘텐츠 해시 기반 파일명으로 저장, 문서엔 상대경로 링크.

## 7. 핵심 흐름

1. **폴더 열기** → Rust 스캔 → 파일 트리(IPC) → 마지막 워크스페이스/문서 복원.
2. **문서 열기** → (미저장 가드) → `read_file` → CM6 버퍼 로드 → Live Preview 렌더.
3. **편집 → 자동저장** → CM6 즉시 디코레이션 렌더 + dirty 표시 → 디바운스/blur 시 → Rust 원자적 저장.
4. **이미지 붙여넣기/드롭** → 프론트 바이트 캡처 → `save_asset` → Rust가 `assets/`에 저장, 상대경로 반환 → `![](assets/…)` 삽입·렌더.
5. **Mermaid/수식/코드** → 블록 진입 시 위젯으로 교체(비동기 렌더 · 디바운스 · 콘텐츠 해시 캐시), 실패 시 인라인 오류 위젯.
6. **내보내기** → 전체 md → 단일 파이프라인 → 자기완결 HTML(폰트/CSS 인라인) / 웹뷰 print-to-PDF.
7. **검색** → `search(query)` → Rust tantivy 인덱스 → 결과(워크스페이스 전역). 인덱스는 watcher와 연동해 갱신.

## 8. 엣지케이스 / 에러 처리

- **외부 수정 충돌:** watcher가 활성 문서 변경 감지 → 로컬 dirty 없으면 조용히 reload;
  있으면 "내 변경 유지 / 디스크 버전 불러오기 / 비교" 배너.
- **저장 실패(권한·디스크 풀):** 원자적 쓰기로 원본 보존 + 사용자 알림 + 재시도 → **데이터 손실 0**.
- **IME / CJK:** 한글 등 조합 입력 중 활성 줄 원본 유지(디코레이션 깜빡임 방지). CM6 IME 경로 별도 테스트.
- **깨진 Mermaid/수식:** 인라인 오류 표시(앱 비충돌).
- **대용량/비정상 파일:** 거대 표·이미지 뷰포트 가상화, 바이너리 오인 파일 가드.
- **미저장 종료/문서 전환:** 가드 + 크래시 대비 자동저장.
- **심볼릭 링크 / 외부 이동·삭제:** 경로 안정성 처리, 파일 사라짐 대응.

## 9. 성능 전략

- **증분 파싱:** CM6 Lezer가 변경분만 재파싱 → 대형 문서도 타이핑 무지연.
- **렌더 디바운스 + 캐시:** Mermaid/KaTeX/Shiki 결과를 블록 콘텐츠 해시로 캐시. 보이는 영역만 렌더.
- **무거운 일은 Rust로:** 검색 인덱싱·폴더 스캔·대용량 IO를 비동기(Rust/tokio)로, UI 블로킹 없음.
- **시작 성능:** 작은 바이너리 + 로컬 번들 폰트(웹폰트 네트워크 의존 0) + 지연 로딩.
- **목표 예산(초안):** 키 입력→렌더 **< 16ms**, 1MB 문서 열기 **< 200ms**, 콜드 스타트 **< 1s**.

## 10. 테스트 전략

- **Rust 코어:** 원자적 저장·경로 처리·검색·asset 저장 단위/통합 테스트(임시 디렉토리 기반).
- **에디터/프론트:** **왕복 무결성**(md → 렌더 → md 동일성) 테스트가 **최우선**(마크다운 손실 0 보장) +
  마크다운/엣지 케이스 스냅샷(Vitest).
- **E2E:** `tauri-driver`로 열기·편집·저장·내보내기·이미지·검색 시나리오.
- **크로스플랫폼 CI:** 3 OS에서 빌드·E2E 실행해 웹뷰 차이 회귀 검출.

## 11. 디자인 시스템 (시작점 — 폴리시 패스에서 정교화)

- **본문 · UI · 제목:** Pretendard (Variable)
- **일본어(かな):** Pretendard JP
- **한자(漢字)·중문:** Noto Sans KR(또는 시스템 CJK) 폴백

  `'Pretendard Variable', Pretendard, 'Pretendard JP', -apple-system, system-ui, 'Apple SD Gothic Neo', 'Noto Sans KR', sans-serif`

### 11.3 색 (시작 팔레트)
- **라이트:** bg `#ffffff` · faint `#f7f8fa` · border `#ecedf1` · text `#181a1f` · muted `#8a909c` · accent(indigo) `#5b5bd6` · accent-soft `#eef0fb`

### 11.4 레이아웃
- **타이틀바:** 미니멀(~40px), 중앙 문서명, 우측 ⌘K 힌트·테마 토글.

- **상태바:** ~28px, 단어 수 · 읽기 시간 · 줄/열.
